Phase 1: New Sensor Simulation Engine (Client-Side JS)
[NEW] src/engine/sensorSimulator.js
Generates realistic real-time streams every 1-2 seconds via setInterval
7 sensors: Engine Temperature (°C), RPM, Brake Pad Wear (%), Tyre Pressure (PSI), Battery Voltage (V), Vibration (g), Speed (km/h)
Each sensor has realistic drift, noise, and bounded ranges based on OBD-II patterns
Injectable fault scenarios with dedicated buttons:
🔥 "Engine Overheating" — temperature ramps 2-5°C/tick
🛞 "Tyre Pressure Drop" — slow bleed from 32 PSI → 15 PSI
🛑 "Brake Wear Critical" — wear jumps from 40% → 90%
🔋 "Battery Drain" — voltage drops from 12.6V → 10V
Data formatted as OBD-II style JSON messages for credibility
[NEW] src/engine/edgeRiskEngine.js
Local threshold + weighted scoring — the "edge AI" that runs with zero cloud dependency
Computes a composite risk score (0-100) from all sensor readings
Per-sensor risk zones: green (0-30), yellow (30-70), red (70-100)
Weighted formula: riskScore = Σ(sensorRisk_i × weight_i) with engine temp and brake wear heavily weighted
Generates instant alerts with severity (INFO / WARNING / CRITICAL) and plain-language descriptions
Time-to-failure estimation based on rate of change: "At current degradation rate, brake failure in ~45 minutes"
All processing is 100% client-side JavaScript — this IS the edge
Phase 2: LLM Copilot Integration (Claude/Anthropic API)
[NEW] src/engine/copilotBrain.js
Ingests the full sensor window every 5 seconds → sends to Claude API
System prompt crafted as an expert automotive mechanic copilot
Context includes: all current sensor values, risk score, recent alerts, rate-of-change trends
Returns: anomaly analysis, predicted failure modes, recommended actions
Auto-analysis: Periodically summarizes vehicle health without user prompting
Chat mode: User can ask natural-language questions like "Is it safe to drive 50km?" and get contextual answers grounded in live sensor data
